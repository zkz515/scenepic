### 3. 坐标系统和变换

#### 相机坐标系变换
- **W2C矩阵**：世界坐标系到相机坐标系的4×4变换矩阵
- **C2W矩阵**：相机坐标系到世界坐标系（W2C的逆）
- **内参矩阵K**：3×3矩阵，包含焦距和主点信息

#### 视锥体计算
```python
hfov = np.rad2deg(np.arctan(W / 2. / K[0, 0]) * 2.)  # 水平视野角
vfov = np.rad2deg(np.arctan(H / 2. / K[1, 1]) * 2.)  # 垂直视野角
```

## 关键参数说明

### frustum_size
- **含义**：控制相机视锥体的大小
- **建议值**：0.01-0.5，根据场景尺度调整
- **影响**：过大会遮挡点云，过小会看不清相机位姿

### 坐标系约定
- **右手坐标系**：确保你的数据使用一致的坐标系
- **相机朝向**：通常Z轴指向前方，Y轴向上
- **尺度统一**：点云和相机位置应该在相同的尺度下

## 可视化结果解读

### 1. 3D散点图
- **彩色点**：RGB点云，颜色反映真实色彩
- **密度**：点的分布反映场景几何结构

### 2. 相机表示

### 3. 空间关系
- **观察角度**：可以看到相机是否正确朝向感兴趣区域
- **覆盖范围**：视锥体是否覆盖了点云的主要部分
- **几何一致性**：图像内容是否与对应空间位置匹配

## 常见问题和解决方案

### 1. 点云显示异常
- **问题**：点云太密集或太稀疏
- **解决**：调整下采样比例或点的大小

### 2. 相机位置错误
- **问题**：相机在奇怪的位置
- **解决**：检查W2C矩阵的计算，确认坐标系约定

### 3. 图像方向错误
- **问题**：图像显示颠倒或镜像
- **解决**：检查图像坐标系和相机坐标系的对应关系

### 4. 尺度问题
- **问题**：相机太大或太小
- **解决**：调整frustum_size参数
